What is Cradle?

Cradle is a tool developed by Magnus Manske for mapping RDF data models to SPARQL queries, making it easier to extract and validate data from RDF datasets.
In simple terms, it helps ensure that the data entered into Wikidata (in this case, inscriptions) follows a consistent structure: for example, that all items include the same kind of information, such as a “findspot” or “place of discovery”.
The key difference is that while ShEx identifies errors after the data has been entered, Cradle helps prevent them in advance.
So, when you query the system for items matching this schema, it highlights anything that does not conform (quality control).
My goal is to build both a ShEx schema and a Cradle form. Currently, I am not able to create a Cradle following Magnus Manske’s instructions, so as an alternative, I am developing a ShEx schema.
The ShEx will differ from EntitySchema:E413, because it is not directly compatible with Cradle (references are not allowed in Cradle), although it will incorporate elements from the data models of both IDEA and Altinum.

Experimental Epigraphic ShEx

This ShEx, currently under development, is based on the IDEA project with its EntitySchema:E413, and Altinum project.

PREFIX wdt: <http://www.wikidata.org/prop/direct/>

  {
    ?item wdt:P31 wd:Q1640824 .   # inscription

  {
    ?item wdt:P31 wd:Q111326612 . # inscribed object

  {

  UNION

  }

  {
